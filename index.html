TiddlyWeb API (2.4.3)

Download OpenAPI specification:

TiddlyWeb is a RESTful web API for managing tiddlers (small pieces of content), bags (containers for tiddlers), and recipes (dynamic queries of tiddlers).

Content Negotiation & URL Extensions

TiddlyWeb supports content negotiation via standard Accept headers or by appending an extension directly to the URL (e.g., /bags/{name}.json or /recipes/{name}.txt). The URL extension takes precedence over the Accept header.

Supported formats:

  • text/html (.html) - HTML representation
  • text/plain (.txt) - Plain text representation
  • application/json (.json) - JSON representation
  • application/vnd.tiddlyweb+json - Strict TiddlyWeb JSON representation

Root page

Returns an HTML landing page that serves as the entry point to the TiddlyWeb API. The page displays a navigation listing with links to the two primary resource collections: /recipes and /bags.

Responses

Default TiddlyWeb server

http://localhost:8080/

List all recipes

Returns the names of all recipes that the current user has permission to read. The response format differs by content type:

  • text/plain: Newline-separated list of recipe names (e.g. "recipe0\nrecipe1\n")
  • application/vnd.tiddlyweb+json: JSON array of recipe name strings (e.g. ["recipe0", "recipe1"])
  • text/html: HTML page with a <ul id="recipes"> listing each recipe as a link to its detail page

Get a recipe

Returns the recipe definition (name, description, policy, and ordered list of (bag, filter) pairs) in the requested format.

The current user must have read permission on the recipe's policy to access it.

The response format differs by content type:

  • text/plain: RFC 822-style headers followed by a newline-separated list of bag entries. Each bag entry is either a bare bag name (for special bags) or a /bags/{bag_name}/tiddlers path, optionally appended with ?{filter_expression}.
  • application/vnd.tiddlyweb+json: A structured JSON object with name, desc, policy, and recipe (the ordered list of (bag, filter) pairs) fields.
  • text/html: An HTML page displaying the recipe's metadata and bag listing.

Request Body schema:
required
string (RecipeText)

Plain text representation of a recipe, using an RFC 822-style header format followed by a blank line and a newline-separated list of bag entries.

Headers (first two lines):

  • desc: — The recipe's human-readable description (may be empty).
  • policy: — The recipe's policy as a JSON-serialized dictionary of access control lists.

Body (after the blank line):

  • Each line is a bag entry. For normal bags, the format is /bags/{bag_name}/tiddlers.
  • For special bags (those with a registered retriever), the bare bag name is used instead.
  • If a filter expression is associated with the bag, it is appended as a query string: /bags/{bag_name}/tiddlers?{filter_expression}.

Responses

Default TiddlyWeb server

http://localhost:8080/recipes/{recipe_name}
